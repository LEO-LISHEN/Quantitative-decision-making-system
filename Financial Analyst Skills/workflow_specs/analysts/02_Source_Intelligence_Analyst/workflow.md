# 工作流

## 第一步：确认分析对象

确认公司名称、股票代码、上市市场、交易所、币种、会计准则、信息截止时间和分析时间范围。  
如果公司身份不明确，先要求补充，不进入后续分析。

如果公司涉及多地上市，例如 A+H、ADR、H 股、红筹或中概股，应标明当前分析使用的主要证券、主要交易市场和主要披露文件。

## 第二步：盘点用户材料

逐项识别用户提供的信息：

- 研报
- 新闻
- 公司公告
- 财务报表
- 电话会纪要
- 行业资料
- 股吧或社交媒体
- K 线图或量价描述
- 用户自己整理的事件和观点

每项材料都要标注来源、日期、类型和可信度。

## 第三步：判断信息缺口

检查是否缺少以下关键材料：

- 最近一期财报
- 最近公告和重大事项
- 最新业绩预告或业绩快报
- 近期新闻和行业变化
- 市场一致预期或研报观点
- 近期股价和成交量反应
- 主要风险事件

如果缺口会影响后续估值，应明确列出。

## 第四步：选择市场信息源

根据上市市场选择优先信息源：

- A 股：巨潮资讯、上交所、深交所、北交所、证监会、互动易、上证 e 互动、公司公告、定期报告、业绩预告、投资者关系活动记录。
- 港股：港交所披露易、香港证监会、公司公告、年报、中报、业绩公告、通函、南向资金、沽空数据。
- 美股和中概股：SEC EDGAR、10-K、10-Q、8-K、20-F、6-K、公司 IR、earnings release、earnings call transcript、Nasdaq/NYSE 信息。

不得把一个市场的披露制度机械套用到另一个市场。

## 第五步：主动搜集信息

如果平台提供搜索、浏览、数据库、插件或 API 工具，应主动补充信息。优先级如下：

1. 公司公告、交易所披露、监管文件
2. 年报、半年报、季报、业绩预告、业绩快报
3. 公司官网、投资者关系记录、业绩会材料
4. 权威财经媒体和行业数据
5. 研报、一致预期、券商观点
6. 股吧、社交媒体和讨论热度

如果平台没有外部工具，不要声称已经搜索。应输出“建议补充的信息”和“建议搜索关键词”。

## 第六步：搜索失败处理

如果搜索不到或无法验证某项关键信息，应输出：

- 已尝试搜索什么
- 未找到什么
- 可能原因是什么
- 建议用户上传或补充什么
- 对后续估值有什么影响

不要用猜测填补关键事实。

## 第七步：建立事件时间线

把重要信息按时间排序，形成事件链。  
重点标注会影响估值的事件：

- 业绩变化
- 管理层指引变化
- 订单和合同
- 产能投放或停产
- 价格变化
- 政策变化
- 并购、回购、分红、融资
- 诉讼、监管、违约、减值
- 股价对事件的反应

## 第八步：证据分级

按以下标准分级：

- A 级，官方强证据：公告、监管文件、审计财报、交易所披露、公司直接披露。
- B 级，权威二级证据：主流财经媒体、行业数据库、公司官网、业绩会纪要、政府和行业协会数据。
- C 级，机构观点：券商研报、评级报告、一致预期、专家访谈。
- D 级，市场情绪：股吧、社交媒体、评论区、投资者讨论。
- E 级，未验证传闻：无法追溯来源的说法、未经确认的截图、二手转述。

## 第九步：提取估值变量

提取会被后续分析师使用的变量：

- 收入增长
- 毛利率和净利率
- 订单和 backlog
- 产能和利用率
- 产品价格和成本
- 资本开支
- 营运资本
- 现金流
- 债务和融资
- 股本变化
- 政策和行业周期
- 催化剂和风险事件

## 第十步：传递给后续模块

根据材料特征，建议调用后续分析师。例如：

- 财报材料丰富：调用财务报表与质量分析师。
- 新闻或公告显示重大事件：调用事件催化剂分析师。
- 市场反应异常：调用市场预期差分析师和量价趋势分析师。
- 股吧讨论密集：调用情绪、舆情与股吧分析师。
- 估值假设风险较大：调用风险、反证与做空视角分析师。
