# 输入结构

## 必填输入

- 公司名称或股票代码
- 上市市场或交易所，如果用户知道
- 用户希望分析的时间范围，例如最近一周、一个季度、一年、上市以来
- 用户已提供的材料，至少一种即可

## 可选输入

- 研报文本或摘要
- 新闻、资讯或链接
- 公司公告、财报、业绩预告、监管问询函
- 电话会纪要、管理层访谈、投资者关系记录
- 股吧、社交媒体、评论区或投资者讨论内容
- 行业数据、宏观数据或政策文件
- 近期日 K 线图、成交量图或用户对图表的描述
- 用户关注的问题，例如估值、风险、催化剂、财报质量、预期差
- 目标市场类型，例如 A 股、港股、美股、中概股、ADR、H 股、A+H 双重上市
- 币种和财务口径，如果用户已知
- 信息截止时间，如果用户希望固定分析时点

## 主动搜集输入

当平台提供搜索或数据工具时，应根据公司名称和股票代码主动补充：

- 最新公司公告
- 最近一期年报、半年报、季报或 10-K、10-Q、20-F 等文件
- 最近业绩预告、业绩快报、投资者关系活动记录
- 近期重大新闻和行业事件
- 主流研报或一致预期信息，如果可获得
- 股价近期重要反应和市场情绪材料

## 输入字段建议

```json
{
  "company_name": "公司名称",
  "ticker": "股票代码",
  "market": "上市市场",
  "exchange": "交易所",
  "currency": "主要币种",
  "accounting_standard": "会计准则",
  "time_range": "分析时间范围",
  "information_cutoff": "信息截止时间和时区",
  "user_materials": [
    {
      "type": "research_report | news | announcement | filing | financial_statement | forum | chart | other",
      "source": "来源名称或链接",
      "date": "发布日期",
      "content": "正文或摘要"
    }
  ],
  "research_focus": "用户重点关注的问题",
  "available_tools": ["web_search", "file_search", "market_data_api", "none"]
}
```

## 缺失输入处理

如果缺少公司名称或股票代码，必须先要求用户补充。  
如果缺少实时搜索工具，必须说明无法主动验证最新信息，并输出建议用户补充的材料清单。
如果上市市场不明确，必须先根据股票代码、公司名称和用户语境判断；仍无法判断时，应要求用户补充市场信息。  
如果币种、单位、会计准则或报告期不明确，必须标注“待确认”，不得直接混合使用。
