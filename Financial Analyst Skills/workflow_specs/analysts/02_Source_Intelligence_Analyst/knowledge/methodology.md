# 方法论

## 1. 信息源优先级

信息源不是平等的。估值工作流应优先使用可追溯、可验证、具有法律或监管责任的信息。

优先级从高到低：

1. A 级，官方强证据：交易所公告、监管披露、公司正式公告、审计财报、定期报告、招股书、监管问询回复。
2. B 级，权威二级证据：公司官网、投资者关系活动记录、业绩会材料、权威财经媒体、行业协会、政府部门、第三方数据库。
3. C 级，机构观点：券商研报、评级报告、一致预期、专家访谈。
4. D 级，市场情绪：股吧、社交媒体、评论区、投资者社区。
5. E 级，未验证传闻：无来源传闻、无法追溯来源的截图、二手转述。

研报可以用于判断市场预期、盈利预测和主流叙事，但不能替代公告和财报来确认事实。

## 2. 事实、观点、预测、传闻

整理材料时必须拆分为四类：

- 事实：已经发生并可由高可信来源验证的信息。
- 观点：分析师、媒体、管理层或投资者对事实的解释。
- 预测：对未来收入、利润、订单、价格、政策或估值的判断。
- 传闻：无法验证来源，或仅在社交媒体和论坛中传播的信息。

## 3. 时效性原则

金融信息有强时效性。整理材料时要标注：

- 信息发布日期
- 事件发生日期
- 财报对应期间
- 市场是否已经对该信息反应
- 信息是否已经被后续公告或财报修正
- 信息截止时间和时区

输出中必须写明：

```text
信息截止时间：YYYY-MM-DD HH:mm，时区
```

## 4. 估值相关性原则

不是所有信息都影响估值。优先提取能够改变以下变量的信息：

- 未来收入
- 利润率
- 自由现金流
- 折现率或风险溢价
- 估值倍数
- 股本数量
- 资产价值
- 负债和流动性
- 催化剂概率
- 市场预期

## 5. 主动搜集原则

如果工具可用，信息源分析师应主动补齐关键缺口，而不是只等待用户上传材料。  
但主动搜集必须遵守三条底线：

- 不编造搜索结果。
- 不把未经验证的信息写成事实。
- 不把低可信来源提升为高可信证据。

## 6. 跨市场适用原则

信息源分析师必须适用于 A 股、港股和美股。  
使用信息前必须确认：

- 上市市场和交易所
- 主要披露平台
- 披露文件类型
- 币种
- 单位
- 会计准则
- 报告期
- 交易时区

常见文件差异：

- A 股：年报、半年报、季报、业绩预告、业绩快报、投资者关系活动记录、问询函。
- 港股：年报、中报、业绩公告、通函、上市文件、权益披露。
- 美股：10-K、10-Q、8-K、20-F、6-K、S-1、DEF 14A、earnings release、earnings call transcript。

## 7. 矛盾处理

当不同来源冲突时，应：

1. 标注冲突点。
2. 比较来源优先级。
3. 检查日期，确认是否存在信息更新。
4. 保留不确定性，而不是强行给出唯一结论。
5. 把关键矛盾传递给风险分析师或总控分析师。

## 8. 搜索失败处理

如果工具可用但没有搜到关键信息，应标准化输出：

- 已尝试搜索的信息
- 未找到的信息
- 可能原因
- 建议补充来源
- 对后续估值的影响

搜索失败不能被沉默处理，因为缺失信息本身就是重要风险。
