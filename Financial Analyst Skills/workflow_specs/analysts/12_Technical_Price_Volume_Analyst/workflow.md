# 工作流

## 目标

用行情数据和事件时间线判断交易行为是否确认投资逻辑，并输出技术确认状态、择时含义和失效信号。

## 步骤 1：确认数据模式

判断输入属于哪种模式：

- 用户上传日 K 表格。
- API 自动获取行情数据。
- 图表截图。
- 文字描述。

如果是图表截图或文字描述，应优先要求结构化日 K 数据；如果无法获得，只能做低置信观察。

## 步骤 2：获取行情和基准数据

需要获取：

- 个股日 K。
- 成交量、成交额、换手率。
- 复权口径。
- 大盘指数。
- 行业指数、主题指数或 ETF。
- 可比公司行情。
- 事件时间线。

如果平台提供 API，应主动调用行情 API。若 API 不可用，应检查用户上传表格是否满足最低字段要求。

## 步骤 3：数据质量检查

检查：

- 是否至少 60 个交易日。
- 是否有成交量。
- 是否有成交额。
- 是否说明复权方式。
- 是否存在停牌、缺失值、异常成交。
- 是否存在除权除息、拆股、配股或合股。
- 基准指数是否匹配公司市场和行业。
- 事件日期是否准确。

输出数据质量评级：高、中、低、不足。

## 步骤 4：趋势状态判断

分析：

- 价格是否处于上升、下降、震荡、筑底、反转、加速或衰竭状态。
- 短期、中期、长期趋势是否一致。
- 最近高点和低点是否抬高或降低。
- 价格是否处于年内高位、低位或中间区间。

## 步骤 5：均线结构分析

分析：

- 5/10/20/50/60/120/200 日均线结构。
- 均线是否多头排列、空头排列、粘合或反复穿越。
- 价格是否站上或跌破关键均线。
- 关键均线是否上行、下行或走平。

如果没有足够历史数据，不要强行计算长期均线。

## 步骤 6：量价关系分析

分析：

- 上涨时是否放量。
- 下跌时是否放量。
- 突破时是否有成交量确认。
- 跌破时是否有恐慌量或持续抛压。
- 缩量横盘是健康整理还是流动性不足。
- 是否存在量价背离。

港股和小盘股必须特别检查成交额质量，低成交额下的技术形态置信度较低。

## 步骤 7：支撑阻力和价格区域

识别：

- 前高和前低。
- 缺口区域。
- 成交密集区。
- 平台整理区。
- 均线支撑或压制。
- 关键失效位。

支撑阻力应表达为区域，不应伪装成精确点位。

## 步骤 8：突破/跌破质量判断

判断：

- 是否有效突破。
- 是否假突破。
- 是否回抽确认。
- 是否有效跌破。
- 跌破后是否快速收复。
- 成交量是否确认价格行为。

突破或跌破必须同时考虑价格延续、成交量、相对强弱和事件背景。

## 步骤 9：相对强弱比较

比较：

- 个股 vs 大盘指数。
- 个股 vs 行业指数。
- 个股 vs 主题 ETF。
- 个股 vs 可比公司。

输出：

- 强于、弱于、持平或无法判断。
- 相对强弱是改善、恶化还是稳定。

## 步骤 10：事件后反应分析

将价格和成交量与事件时间线对齐：

- 财报后上涨或下跌。
- 利好不涨。
- 利空不跌。
- 放量跳空。
- 消息前提前异动。
- 消息后冲高回落。
- 消息后缩量横盘。

注意：这一步与市场预期差分析师有关，但本分析师只输出交易反应，不独立判断全部预期差。

## 步骤 11：形成技术确认结论

输出确认状态：

- 强确认。
- 弱确认。
- 中性。
- 预警。
- 反向否定。
- 无法判断。

同时给出：

- 择时含义。
- 技术失效信号。
- 需要补充的数据。
- 需要其他分析师复核的问题。

## 低数据场景处理

如果只有很少行情数据：

- 明确说明数据不足。
- 不输出高置信趋势结论。
- 不判断长期均线。
- 不判断相对强弱。
- 只列出可观察到的短期交易反应。

如果只有截图：

- 可描述可见图形，但必须提醒截图可能存在比例、复权和时间范围问题。
- 要求补充结构化日 K 表格或 API 数据。
